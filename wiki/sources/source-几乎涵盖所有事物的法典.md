## Why It Matters

- 这篇文章是 OpenAI 对 Codex 的一次产品级总更新，说明 Codex 的定位已经从“写代码工具”扩展到“能协同电脑、应用和工作流的持续执行伙伴”
- 文章把计算机控制、内置浏览器、图像生成、插件、自动化和记忆功能放在同一条产品叙事里，适合当作后续 Codex 相关页面的上游来源
- 对这个知识库来说，它是理解 Codex 产品能力边界变化的直接来源，而不只是单点功能说明

## Key Claims

- Codex 可以在后台运行的电脑上使用自己的光标执行查看、点击和输入等操作，多个代理可以并行工作
- Codex 现在可以与 Web 原生交互，并在内置浏览器里直接做页面注释，向代理提供更精确的指令
- Codex 支持使用 `gpt-image-1.5` 生成和迭代图像，覆盖产品概念、前端设计、模型和游戏视觉
- 新增了 90 多个插件，扩展了跨工具收集上下文和采取行动的能力
- 开发者工作流支持继续增强，包括 GitHub 评论、多个终端标签页、SSH 远程环境、侧边栏文件打开，以及 PDF、表格、幻灯片和文档的富预览
- 自动化能力支持重用线程、保留上下文、安排未来工作，并在 Slack、Gmail、Notion 等工具中跟进长期任务
- 记忆功能预览允许 Codex 记住偏好、更正和曾经收集的信息，并据此推荐下一步工作
